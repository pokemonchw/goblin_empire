# 哥布林帝国

`哥布林帝国` 是一个静态浏览器增量/放置经营游戏原型。项目以 `https://bitbucket.org/bloodrizer/kitten-game` 的源码结构作为参考对象，但题材、文案、资源链和长期循环按 `策划案/` 中的中文设计文档重新设计。

当前可运行实现位于 `game/`，目标是在无需构建工具的前提下验证地穴生存、资源生产、职业分配、研究、工坊、外交、祭祀、远征、威望重置和挑战模式等核心循环。

## 快速开始

直接用浏览器打开：

```text
game/index.html
```

## 目录结构

```text
.
├── README.md
├── AGENTS.md            # 指挥AI工作时的信息，主要是基本项目信息，代码风格和规范约束
├── game/                # 哥布林帝国当前静态浏览器实现
├── 策划案/              # 权威设计文档和实现同步基线
├── 蓝图/                # 界面蓝图页面
└── 开发计划/            # 开发 TODO 与阶段计划
```

`game/` 内部按职责拆分：

- `index.html`：应用入口和脚本加载顺序。
- `data/`：稳定 ID、静态定义和中文显示文本。
- `scripts/`：数据契约和主启动逻辑。
- `state/`：初始状态、存档、加载和迁移。
- `systems/`：资源、建筑、人口、职业、科技、外交、掠夺、祭祀、远征、威望、挑战和模拟循环。
- `ui/`：渲染和界面事件绑定。
- `styles/`：管理界面样式。
- `tests/manual-regression.md`：手动回归脚本。

## 设计文档

修改机制前先阅读 `策划案/00_策划案索引.md`，再进入对应专题文档：

- `策划案/01_猫国建设者源码机制分析.md`：参考实现的系统结构。
- `策划案/02_哥布林帝国核心策划案.md`：世界观、阶段结构和核心体验。
- `策划案/03_资源建筑职业系统.md`：资源、建筑、职业和生产链。
- `策划案/04_科技外交信仰与重置.md`：科技、外交、祭祀、威望和挑战。
- `策划案/05_界面交互与开发落地规格.md`：UI、存档形状、解锁规则、公式和里程碑。

当策划案中的旧建议与 `game/` 当前实现冲突时，以文档中的“当前实现同步基线”和运行时代码为准。

## 开发约定

- 内部 ID 使用稳定 ASCII，例如 `fungus`、`mud_hut`、`chief_hall`；玩家可见文本使用中文。
- 机制数据应保持数据驱动，资源、建筑、科技、职业、解锁和价格不要写成 UI 字符串。
- 存档只保存进度和状态，不复制完整静态定义；修改存档结构时同步更新迁移逻辑。
- 新增或重构代码时遵守 `AGENTS.md` 中的注释、命名、数据契约和格式要求。

## 验证

当前没有自动化测试。改动后至少执行窄范围人工验证：

1. 打开 `game/index.html`。
2. 按 `game/tests/manual-regression.md` 中与改动相关的流程检查。
3. 验证暂停/继续、保存/加载、导出/导入不会破坏当前进度。
4. 如果改动了存档结构，额外验证旧版本存档迁移到当前 `SAVE_VERSION`。
