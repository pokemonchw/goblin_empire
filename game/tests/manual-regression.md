# 哥布林帝国手动回归脚本

## 纯函数与核心规则最小验证

- 价格计算：在浏览器控制台或 Node VM 中对同一建筑调用 `game.buildings.getBuildingPrice(state, definition)`，确认拥有数增加后价格按 `priceRatio` 指数增长，威望天赋 `cave_engineering` 会降低价格。
- 资源支付：准备资源不足和资源足够两种状态，分别调用 `game.resources.canAfford` 与 `game.resources.spendResources`，确认不足时状态不变，足够时只扣除价格数组中的资源。
- 解锁判断：对资源、建筑、科技和标签页分别设置刚低于阈值与达到阈值的状态，调用 `game.unlocks.applyUnlockBundle` 后确认只在达到条件时解锁。
- 职业产出：创建已解锁职业和具体哥布林，调用 `game.jobs.assignWorker` 与 `game.jobs.updateJobs`，确认资源每秒变化和职业技能经验同步增长。
- 技能等级：给哥布林写入不同技能经验，确认职业适配和远征评分随技能经验提高。
- 苗床繁育：创建普通俘虏并调用 `game.captivesSystem.applyDisposition(state, captiveId, "bed")`，确认俘虏进入 `breedingState = "gestating"` 且不会弹确认框；推进一个月后按失败概率结算，成功时生成 1 个 `origin = "captive_bed"` 的新哥布林，随后进入 `resting`；休养期间确认 `培育新生` 不可执行，但 `洗脑改造` 和 `做成食物` 可执行；再推进一个月后回到 `idle`，同一俘虏可再次培育；随后调用 `game.population.updatePopulation`，确认不会因空住房和菌菇自动新增人口。
- 战兽：调用 `game.warbeastsSystem.createWarbeast("cave_boar", "manual_test", state)` 并加入 `state.warbeasts`，确认口粮口数增加；连续 4 次 `applyDisposition(state, warbeastId, "tame")` 后可执行 `breed`；推进一个月后生成 1 个 `origin = "warbeast_bed"` 的新哥布林并进入 `resting`，休养期间 `calculateFoodConsumerUnits` 翻倍；执行 `butcher` 后战兽移除且菌菇固定 +100。
- 战兽随队与触手怪：创建并驯化任意战兽，在掠夺地点选择该战兽，确认预览队伍强度增加物种 `raidStrength`、任务保存 `warbeastId`，且返程前不能被另一支队伍选择；将触手怪随队并强制成功结算，确认新俘虏含 `tentacle_broodbed`，由该俘虏成功繁育的新生含 `bloodline_tentacle_monster`，原神灵血脉不被覆盖；分别固定随机数低于和等于 `0.001`，确认任意地点只有低于阈值时捕获触手怪。
- 满仓口粮：设置 1 个存活哥布林、5 个启用 `fungus_bed`、菌菇达到容量上限，调用 `game.simulation.updateGame` 推进 60 秒，确认不会产生断粮计时或饿死日志；再设置无菌菇、无菌菇产出且有俘虏，推进超过 3 个游戏日，确认真实断粮会优先移除俘虏并每名返还 10 菌菇；俘虏耗尽后继续断粮才会杀死哥布林。
- 日期速度：调用 `game.calendar.updateCalendar(state, 1)` 后，确认 `state.calendar.elapsedDays` 增加 1；调用 `game.calendar.calculateDaysFromSeconds(30)` 后，确认返回 30。
- 月末自动存档：准备未暂停状态并记录 localStorage 原存档，将 `state.calendar.elapsedDays` 设为 29 后调用 `game.simulation.updateGame(state, state.lastActiveTimestamp + 1000)`，确认 `localStorage.getItem("goblinEmpireSave")` 被更新且存档中的 `calendar.elapsedDays >= 30`；再将 `state.isPaused` 设为 `true` 后等待或调用 `updateGame`，确认不会因为暂停推进而写入新月末存档。
- 暂停门控：将 `state.isPaused` 置为 `true` 后调用 `game.simulation.updateGame`，确认返回 `0` 且资源、人口、事件、自动制作和远征不推进。
- 存档迁移：用 v1 存档文本调用 `game.save.loadFromText`，确认版本迁移到当前 `SAVE_VERSION`，缺失的契约、远征、挑战、威望、日期和开局俘虏字段被补齐。

## 手动回归流程

1. 新局到第一个窝棚：从空存档开始，采集菌菇、朽木和碎石，建造第一个 `菌菇床` 与 `泥棚`。
- 手动采集科技树专项：分别研究采菌、捡朽木、搬碎石的四级分支，确认对应按钮单次显示与实际产出依次为 1、2、3、3、5，且不会串改另外两个按钮。
- 手动采集事件池专项：新局未研究采集科技时反复点击三个按钮，确认每个按钮都能触发自己的 5 个事件池；完成第二、第四级科技后正面事件出现比例提高，完成第三级安全科技后对应两个负面事件不再出现。资源满仓时继续点击，仍应有 15% 概率投掷事件。
2. 解锁氏族页并分配职业：通过开局俘虏培育新生扩张人口，建造解锁氏族页所需建筑，分配和撤下至少一个职业。
3. 解锁研究页并研究采菌：建造 `涂鸦墙`，进入研究页，研究 `采菌`，确认解锁职业循环。
- 研究图谱专项：新局建造涂鸦墙后确认“刻痕”为唯一可研究入口，直接后继以预览态显示；完成刻痕后确认早期节点按生存繁衍、矿业工程等路线和四时代列分组，不再混成行列表；采菌、捡柴、搬石三条专精默认各折叠为一张链卡，并可展开/收起四项真实节点；展开节点必须紧跟所属专精总卡，不得排到“刻痕”或其他普通科技卡片之后。
- 研究连线专项：在“全部状态”的路线视图中确认每个可见子节点均由曲线实际连接到其可见前置卡片边缘；同列节点使用纵向连接，跨时代或跨路线节点连接到真实父节点，任选前置使用虚线；切换状态筛选后，不得为已隐藏的父节点保留悬空短线。
- 研究状态专项：分别构造前置未完成、资源不足、资源足够、已完成和里程碑节点，确认卡片同时使用图形、文字和边框表达 `preview`、`unaffordable`、`available`、`researched` 与里程碑权重；暂停时可研究节点按钮显示“已暂停”。
- 研究分类滚动专项：桌面端分别拖动至少两条科研路线底部的横向滚动条，确认各路线滚动位置互不影响、左侧路线介绍始终不被卡片覆盖，且筛选或研究触发重绘后各路线恢复原滚动位置；同时确认横向滚动条、页面右侧纵向滚动条和科技搜索框符合地穴控制台主题。
- 研究分类空白拖动专项：当路线内容宽于分类卡片时，在路线介绍或时代列的空白处按住鼠标左键左右拖动，确认该分类底部滚动条同步移动且其他分类不动；从科技卡片、按钮、输入框或链接上按下时不得启动拖动。
- 研究分类拖动保持专项：拖动任一科研路线后松开鼠标，再把鼠标移入该路线的科技卡片并等待自动界面刷新，确认底部滚动条不跳回最左侧，科技连线仍与恢复后的节点位置一致。
- 研究详情专项：使用鼠标悬停科技卡片及键盘聚焦卡片，确认详情浮窗显示介绍、瓶颈建议、等待时间、前置、解锁内容、后继和状态；移出或失焦后浮窗隐藏，卡片内按钮仍可独立操作且不再出现“查看详情”按钮。
- 研究关联专项：选择同时拥有已揭示前置与后置的研究卡片，确认显示“查看关联”按钮；点击后前置列表位于按钮左侧、后置列表位于右侧，浮框保持半透明且不被图谱滚动区域裁切；点击页面空白处自动关闭，重新打开并点击任一关联项后页面平滑滚动到对应卡片且浮框关闭；搜索、路线聚焦或列表视图下跳转时应先恢复完整图谱，未知节点名称不得被关联列表提前泄露。
- 研究路线专项：完成采矿后确认冶金、粗制工具、兽栏/弓弩和氏族规矩分属正确路线；完成氏族规矩后中期节点仍按路线分流。点击路线徽签只聚焦路线，再次点击恢复全局；“定位可研究”聚焦图谱顺序中的最早可研究节点。
- 研究旧档专项：载入已完成后期科技的旧存档，确认研究完成标记保持、六路线进度正确、已研究路径仍显示，科技价格、解锁包与迁徙入口行为没有变化。
4. 保存、刷新、加载：保存到 localStorage，刷新页面后加载，确认暂停状态、资源、建筑、哥布林和日志可恢复。
5. 暂停、等待、继续：暂停后等待至少 10 秒，确认资源和远征不推进；继续后恢复正常模拟。
6. 导出、导入：导出存档文本到顶部文本框，粘贴同一文本并点击 `载入文本`，确认当前状态被替换且不出现浏览器弹窗。
7. 工坊批量制作：解锁工匠棚和配方后分别制作 `1`、`10`、`100`、`全部`，确认成本和产物正确。
8. 掠夺成功和失败：分配足够抢掠兵后输入派出人数，分别执行低风险和高风险掠夺，确认收益、关系下降、伤亡或失败日志。
9. 俘虏处置：获得俘虏后分别测试培育新生、洗脑改造、做成食物，确认收益、状态变化和风险日志；俘虏卡片浮窗显示洗脑程度、孕育成功率、孕育失败率、逃跑率、反抗率和洗脑消耗，洗脑改造每次固定扣除 100 菌菇，菌菇不足或洗脑程度达到 100% 时按钮置灰或执行失败；俘虏处于休养状态时，洗脑改造和做成食物仍可执行，做成食物会移除该俘虏。
10. 祭祀升级：购买祖灵升级并执行献祭，确认祖灵回响、风险和具体哥布林影响。
11. 工程师自动制作：选择自动制作配方，分配工程师，确认自动制作进度和产物增加。
12. 远征结算：启动普通路线和高风险路线，确认暂停不推进，完成后记录成员、路线、结果和损失。
13. 帝国迁徙：研究迁徙法典并打开深渊门，预览帝国遗产，点击迁徙后直接执行，检查保留和清除项且不出现浏览器弹窗。
14. 挑战模式约束：新局选择每个挑战，确认对应规则生效；完成挑战后确认永久奖励跨局生效。

## 版本范围与节奏记录

- 版本一地穴求生：覆盖手动采集、菌菇床、窝棚、储物坑、涂鸦墙、氏族页、研究页、基础职业和暂停/保存。
- 版本二矿坑氏族：覆盖浅矿井、粗熔炉、工匠棚、基础配方、职业技能、事故、俘虏和工坊操作。
- 版本三地底城邦：覆盖酋长厅、政策、黑市、外交、掠夺、祭祀、账房和行政统计。
- 版本四黑铁帝国：覆盖钢铁、机械、符文、黑铁要塞、深炉、工程师自动制作、高阶外交和战争目标。
- 版本五深渊远征：覆盖深渊门、远征路线、契约、威望迁徙、挑战模式、重置保留和跨局奖励。
- 0-60 小时节奏：帝国页 `阶段节奏校验` 面板显示 0-1 分钟、1-5 分钟、5-15 分钟、15-30 分钟、30-60 分钟、1-3 小时、3-8 小时、8-20 小时、20-60 小时目标。
- 多周目标：威望天赋、挑战完成标记、帝国遗产和重置后奖励保留构成长线循环；每次平衡调整后至少执行一次迁徙和一次挑战完成检查。
