# 界面交互与开发落地规格

## 1. 信息架构

游戏建议采用与源码类似的文本管理界面，但替换为“地穴控制台”风格。

左侧：

- 资源列表：显示当前值、上限、每秒变化。
- 重要状态：哥布林数量、空闲个体、住房、服从度、当前天气、天气影响、拥挤度、当前日期、tick 频率、历法速度、运行状态。
- 资源分组：基础、加工、稀有、神秘、威望。

中间：

- 当前标签页内容。
- 建筑行列表、科技、职业、工坊、外交等操作列表。
- 建筑行列表采用类似资源卡片的纵向行布局，每一行显示建筑名称、拥有数量、本次所需资源、缺失资源、可用倒计时和建造按钮；建筑详情通过整行悬浮框展示。
- 研究列表采用和建筑列表一致的纵向行布局，每一行显示研究名称、线路、研究消耗、缺失资源、可用倒计时和研究按钮；研究详情通过整行悬浮框展示。
- 其他操作按钮显示成本、效果、是否可购买、缺少资源。

右侧：

- 日志消息；每条日志必须带日期前缀，未研究历法时显示季节日，研究历法后显示哥布林历日期。
- 事件提示。
- 自动化/过滤/隐藏已完成开关。

全局工具栏：

- 暂停/继续：显示当前运行状态；暂停时按钮文案为“继续”，继续时按钮文案为“暂停”。
- 保存/加载：暂停状态下仍可使用，并且保存数据必须记录当前是否暂停。
- 作者信息、GitHub 地址、捐款入口：作为项目级链接保留在全局工具栏。

新局模式入口：

- 没有本地存档、重置存档后、或帝国迁徙生成的未初始化新局，视为游戏未开始；地穴标签页正文只能显示“地穴控制台”和“新局模式”两个卡片，不显示地穴剖面、手动采集、基础建筑或其他已开始后的操作卡片。
- “新局模式”卡片内部必须同时提供“正常模式”和“挑战模式”入口，不能只显示挑战模式，也不能把每个挑战拆成独立页面卡片。
- 正常模式入口文案为“开始正常模式”；选择后记录本局模式，不启用挑战规则，之后按标准地穴生存流程推进。
- 挑战模式入口在同一“新局模式”卡片内列出可选挑战按钮；选择后记录本局模式和挑战 ID，并立刻应用挑战规则。
- 选择正常模式或任一挑战后即视为开局；地穴标签页不再显示“新局模式”卡片，改为展开地穴剖面、手动采集、基础建筑等正式地穴内容。已有资源、人口、建筑或科技进度的旧普通局也不显示新局入口。

## 3. 交互规则

建筑行列表：

- 可购买：正常显示。
- 资源不足：置灰，显示缺口。
- 未解锁：隐藏。
- 接近解锁：可在达到 `unlockRatio` 后显示灰态提示。
- 每个已显示建筑占一行，行内基础信息固定为建筑名称、拥有数量、所需资源、缺失资源、可用倒计时和操作按钮组。
- 鼠标停留或键盘聚焦建筑整行时显示悬浮框；不能只让建筑名或按钮触发。
- 建筑悬浮框必须包含建筑介绍、当前建筑成本、摧毁返还、建筑效果和价格倍率；后续如果加入开关、阶段或维护费，也应继续放在悬浮框详情中。
- 拥有数量必须直接读取 `BuildingState.owned`，不能从按钮文本或 DOM 推导。
- 所需资源显示按当前拥有数量和价格倍率计算后的本次建造成本。
- 缺失资源必须在所需资源的同一行信息区域内显示，不能放在建造按钮下方。
- 可用倒计时在当前可支付时显示“可用”，资源不足但按当前每秒产量可达时显示预计等待时间，不可达时显示“不可用”。
- 建造按钮放在行尾操作按钮组内；暂停、未解锁或资源不足时必须置灰。
- 当 `BuildingState.owned > 0` 时，建造按钮旁必须显示“摧毁”按钮；每次只摧毁 1 个同 ID 建筑，摧毁按钮暂停时置灰。
- 摧毁建筑返还最后一座同 ID 建筑按当前价格曲线估算成本的 1/10；返还资源仍受当前容量限制，建筑拥有数、启用数、容量类效果和劳力占用必须同步更新。建筑造成的一次性解锁、成就和统计标记不回滚。

研究行列表：

- 可研究：正常显示，研究按钮可点击。
- 资源不足或暂停：置灰，研究按钮不可点击，并在行内显示缺失资源和可用倒计时。
- 已研究：继续显示完成态或被“隐藏已完成”筛选隐藏；显示时研究按钮为“完成”且不可点击。
- 未解锁：隐藏。
- 每个已显示研究占一行，行内基础信息固定为研究名称、线路、研究消耗、缺失资源、可用倒计时和研究按钮。
- 鼠标停留或键盘聚焦研究整行时显示悬浮框；不能只让研究名称或按钮触发。
- 研究悬浮框必须包含研究介绍、线路、研究消耗、解锁内容和研究状态；后续如果加入前置科技、研究时间或排队系统，也应继续放在悬浮框详情中。
- 研究消耗必须直接读取 `TechnologyDefinition.price`，不能从按钮文本或 DOM 推导。
- 缺失资源必须在研究消耗的同一行信息区域内显示，不能放在研究按钮下方。
- 可用倒计时在当前可支付时显示“可用”，资源不足但按当前每秒产量可达时显示预计等待时间，不可达时显示“不可用”，已研究时显示“已完成”。
- 研究按钮放在行尾；暂停、已研究、未解锁或资源不足时必须置灰。

资源显示：

- 资源为 0 且未解锁时隐藏。
- 资源首次获得后永久显示。
- 每秒变化为正显示绿色，为负显示红色。
- 容量满时高亮提示。
- 鼠标停留或键盘聚焦资源卡片时显示悬浮框，悬浮框必须包含总产出速度、产出明细、加成明细、总消耗、消耗明细、buff 明细、最终产出和库存爆仓时间。劳力资源卡片例外：不显示总消耗、消耗明细、最终产出和库存爆仓时间，必须额外显示劳力来源、建筑占用和生产状态；劳力来源只显示存活哥布林派生值；建筑占用至少包含每类启用建筑的启用数量、单座占用、减免后合计占用和当前总占用减免比例；生产状态必须明确提示劳力过载时除菌菇床外建筑停产。
- 资源悬浮框只统计当前瞬时每秒流量：职业、建筑自动生产、建筑转换、哥布林与俘虏口粮消耗、契约代价和工程师自动制作预估应计入；手动采集、贸易、掠夺、远征结算、随机事件即时奖励等非持续收益不计入。
- 总产出速度为所有正向持续来源之和，总消耗为所有负向持续来源的绝对值，最终产出为总产出速度减总消耗；如果资源处于暂停状态，悬浮框保留最近一次流量并在 buff 明细中标注“已暂停，当前不推进”。
- 产出明细和消耗明细必须显示来源名称与每秒数量；加成明细显示影响该资源持续流量的建筑、科技、政策、祭祀、契约、天气、威望、挑战、事件或工坊升级倍率；buff 明细显示临时事件、契约代价、自动制作选择、容量满、断粮等当前状态提示。
- 库存爆仓时间使用当前库存、容量和最终产出计算：容量有限且最终产出大于 0 时显示到达上限的预计时间；已满显示“已爆仓”；无容量上限、最终产出不大于 0 或资源未显示时显示“不会爆仓”。

职业分配：

- 每个职业有 `+`、`-`、`全部撤下`、`最大分配`。
- 支持职业预设：生存、采矿、研究、战争、祭祀。
- 人口不足时按钮置灰。
- 批量按钮只改变底层哥布林对象的 `jobId`；职业人数必须从当前存活哥布林统计得出。
- 自动分配默认优先选择对应技能和属性更高的空闲哥布林，已固定职业的 `isPinned` 个体不被预设覆盖。
- 人口普查显示姓名、年龄、总寿命、寿命拆分、当前职业、核心属性、特质、前三项技能、伤病、领袖标记，并支持按职业、特质和伤病筛选。
- 单个哥布林行支持撤下职业、固定/取消固定、设为领袖、查看详情；大规模人口只展示优先列表和筛选结果，避免一次渲染全部对象。

手工制作：

- 支持制作 1、10、100、全部。
- 显示制作收益倍率。
- 后期工程师可自动制作指定配方。

暂停/继续：

- 暂停按钮为全局常驻控件，不随标签页隐藏；所有阶段和所有存档都可使用。
- 暂停后主循环不再推进资源、人口、职业经验、事件、自动制作、掠夺、远征和临时效果倒计时。
- 暂停后仍允许浏览界面、切换标签页、调整日志过滤、保存、加载、导出、导入和重置存档。
- 主循环每跨过 30 个游戏日视为经过 1 个游戏月，必须在本 tick 全部模拟结算、灭亡检测和 `lastActiveTimestamp` 更新后自动写入一次本地存档；自动存档复用普通 `SaveData` 结构，不新增静态定义副本或独立槽位。
- 暂停期间日期不推进，因此不会触发月末自动存档；继续后只从继续时刻向后计算，不补写暂停期间的月份。
- 如果本 tick 因哥布林和俘虏全灭而清空本地存档并回到新局入口，则跳过该 tick 的月末自动存档，避免把灭亡后的新局立即写回本地存档。
- 顶栏必须提供“重置”按钮；点击后先弹出确认，确认后清空当前本地存档并创建全新 `GameState`，不保留资源、建筑、科技、哥布林、威望、挑战、统计或日志。
- 顶栏“重置”是普通存档清空，不等同于帝国迁徙/崩塌威望重置；它不发放帝国遗产，不触发任何长期保留收益。
- 暂停后会改变游戏状态的主动操作必须置灰或拦截，包括手动采集、购买建筑、研究科技、制作配方、分配职业、贸易、掠夺、献祭、政策切换和帝国迁徙。
- 继续后从暂停瞬间的状态恢复，不结算暂停期间的离线收益、事件或消耗。
- 页面关闭或刷新时如果存档处于暂停状态，重新载入后必须继续保持暂停状态。
- UI 必须在状态栏或顶栏显示“运行中/已暂停”，并在已暂停时用日志或提示说明生产已停止。

外交与掠夺：

- 外交页先显示“外交 / 掠夺”二级标签；每个二级标签内再显示世界子标签（地底、地表、深渊）和势力子标签，当前页面只展示所选势力下的地点。
- 贸易和掠夺地点行必须显示距离、在途数量、成本、收益或风险摘要；悬浮框必须补充返程时间、声名门槛、缺口和结算说明。
- 发起贸易时立即扣除贸易成本，并按地点距离生成在途贸易队；返程完成后才结算收益、关系和善名。
- 发起掠夺时立即扣除每名战斗哥布林 100 菌菇，并按目标距离生成在途掠夺队；返程完成后才结算成功/失败、伤亡、关系、恶名、善名、奖励和俘虏。
- 在途掠夺队的成员返程前不能被再次派出掠夺；暂停时贸易队、掠夺队和深渊远征的倒计时都不推进。

外交/掠夺：

- 贸易按钮显示消耗、可能收益、成功概率和关系变化。
- 外交标签页解锁时必须自动显示抢掠相关的大木棒研究，避免掠夺入口被铁片阶段的弓弩卡住。
- 掠夺不能混入研究卡片；外交页固定提供“外交 / 掠夺”两个子标签，掠夺子标签在抢掠兵或战争头目职业已解锁后可进入。
- 外交和掠夺子标签内的所有地点必须通过“地底 / 地表 / 深渊”世界子标签切换；进入某个世界后，再通过势力子标签切换该世界下的地点。
- 地点列表必须使用与资源列表相同的紧凑行形式；行内只放地点名、核心数值、缺口摘要、派出人数按钮组或操作按钮。
- 地点完整信息必须通过鼠标悬停或键盘聚焦浮窗展示：外交地点显示说明、关系、消耗、收益、成功率、善名门槛和缺口；掠夺地点显示说明、队伍强度、目标强度、成功率、伤亡概率、关系下降、报复可能、声名变化、收益和俘虏范围。
- 掠夺地点行必须提供与职业分配相同的派出人数按钮组：`-`、`+`、`最大分配`；仅允许当前已分配的抢掠兵和战争头目参与，最低值不低于目标最低队伍人数，最高值不高于可派出战斗职业人数。掠夺不消耗劳力，也不消耗独立战斗资源；行内和浮窗必须显示菌菇成本，按每名派出哥布林 100 菌菇计算，菌菇不足时按钮置灰并显示缺口。
- 外交页不显示俘虏列表或俘虏处置卡片；俘虏管理由氏族页承载。
- 氏族页俘虏列表显示类型、质量、来源、洗脑程度、繁育状态、可选处置、培育新生收益、改造收益、食物收益、逃脱风险和报复风险；完成“欲望启蒙”后，每张俘虏卡显示该俘虏自己的“自动洗脑：开/关”按钮；完成“公用苗床”后，每张俘虏卡显示该俘虏自己的“自动培育：开/关”按钮；俘虏详情浮窗必须显示年龄、总寿命、寿命拆分、洗脑程度、自动洗脑状态、自动培育状态、自动培育排序值、洗脑固定消耗、孕育成功率、孕育失败率、逃跑率、反抗率、继承倾向概率和状态；俘虏处置按钮不弹二次确认，风险必须在卡牌内提前展示。

## 4. 核心数据结构建议

### Resource

```ts
type Resource = {
  id: string;
  name: string;
  value: number;
  maxValue: number;
  visible: boolean;
  craftable?: boolean;
  persists?: boolean;
  calculatePerTick?: boolean;
  category: "basic" | "crafted" | "rare" | "mystic" | "prestige";
};
```

### Building

```ts
type Price = {
  resource: string;
  amount: number;
};

type Building = {
  id: string;
  name: string;
  description: string;
  owned: number;
  active: number;
  unlocked: boolean;
  prices: Price[];
  priceRatio: number;
  effects: Record<string, number>;
  unlocks?: UnlockBundle;
  stage?: number;
  stages?: BuildingStage[];
  togglable?: boolean;
};
```

### Technology

```ts
type Technology = {
  id: string;
  name: string;
  description: string;
  researched: boolean;
  unlocked: boolean;
  prices: Price[];
  unlocks: UnlockBundle;
};
```

### Job

```ts
type Job = {
  id: string;
  name: string;
  unlocked: boolean;
  modifiers: Record<string, number>;
  skillId: string;
  attributeWeights: Partial<Record<AttributeId, number>>;
};
```

说明：`Job` 是职业定义和解锁状态，不保存权威分配人数。职业人数、空闲人数和职业产出都由 `goblins` 中存活个体的 `jobId` 统计得到。

### Goblin

```ts
type AttributeId =
  | "strength"
  | "dexterity"
  | "cunning"
  | "perception"
  | "will"
  | "attunement";

type SkillId =
  | "foraging"
  | "woodcutting"
  | "hauling"
  | "mining"
  | "smelting"
  | "crafting"
  | "raiding"
  | "scribing"
  | "ritual"
  | "overseeing";

type Goblin = {
  id: string;
  name: string;
  nickname?: string;
  faithId: string | null; // null 表示祖灵祭坛建立前没有信仰；建立后为 goblin_ancestor
  age: number; // 年，允许小数；每个生存月增加 1/12
  baseLifespanYears: number;
  growthLifespanYears: number;
  technologyLifespanYears: number;
  eventLifespanYears: number;
  elderDeathCheckCount: number;
  origin: "natural" | "captive_bed" | "warbeast_bed" | "migrant" | "vassal" | "event" | "legacy"; // natural 仅用于兼容旧存档记录，不再由当前规则生成
  jobId?: string;
  attributes: Record<AttributeId, number>;
  traits: string[];
  skills: Partial<Record<SkillId, number>>;
  wounds: string[];
  isLeader: boolean;
  isPinned: boolean;
  isAlive: boolean;
};
```

说明：每个哥布林都必须是唯一对象。`id` 是保存、日志、死亡记录、领袖引用和职业固定的稳定键；`name` 等显示字段允许调整文案，但不允许作为引用键。`faithId` 在祖灵祭坛建立前为 `null`；首次建立祖灵祭坛后，现有哥布林和之后出生的新哥布林都写入 `goblin_ancestor`，显示为“哥布林祖灵”。`age` 以年为单位，每跨过一个 30 游戏日的生存月增加 `1/12` 年。总寿命 = `baseLifespanYears + growthLifespanYears + technologyLifespanYears + eventLifespanYears`；达到总寿命后每月 1 号投老死骰，首次 10%，每次未死后 `elderDeathCheckCount +1`，下月概率 +10%。`attributes` 的六个 key 必须全部存在，数值建议为 1-10 的整数。`skills` 只保存大于 0 的技能经验，缺省按 0 处理。

type GoblinNamePool = {

说明：姓名池独立于存档。新增名字只能追加或通过迁移处理，避免已保存哥布林的显示名随机变化。

## 5. 解锁系统

采用统一 `UnlockBundle`：

```ts
type UnlockBundle = {
  tabs?: string[];
  resources?: string[];
  buildings?: string[];
  jobs?: string[];
  technologies?: string[];
  upgrades?: string[];
  crafts?: string[];
  policies?: string[];
};
```

解锁时机：

- 研究科技后立即解锁 `unlocks`。
- 建造建筑后解锁建筑绑定内容。
- 首次获得某资源后解锁相关显示。
- 首次拥有 1 只存活哥布林后解锁外交标签页，并立即解锁 `big_club` 大木棒研究；旧存档在加载或下一次模拟推进时同步补齐。
- 达到某资源上限或数量阈值后解锁隐藏建筑。

接近解锁：

```text
如果玩家拥有资源 >= 建筑基础成本 * unlockRatio，则显示该建筑的灰态按钮。
```

## 6. 关键公式

主循环门控：

```text
if isPaused:
  deltaTime = 0
  skip production, consumption, population growth, events, automation, expeditions
else:
  deltaTime = now - lastActiveTimestamp
  run simulation with deltaTime
  lastActiveTimestamp = now
```

说明：暂停是全局时间门控，不是把各系统产出倍率设为 0。这样可以避免某些系统继续扣资源、推进倒计时或触发事件。

建筑成本：

```text
cost(resource) = baseCost(resource) * priceRatio ^ owned
```

资源每秒：

```text
perSecond = ticksPerSecond * (
  basePerTick
  + sum(goblinJobOutput)
  + buildingFlatOutput
  + autoProduction
) * (1 + ratioBonus + globalBonus) * obedienceModifier * seasonModifier * weatherModifier
```

单个哥布林职业产出：

```text
goblinJobOutput = jobBaseOutput
  * attributeModifier(goblin.attributes, job.attributeWeights)
  * skillModifier(goblin.skills[job.skillId])
  * woundModifier(goblin.wounds)
  * leaderModifier
attributeModifier = clamp(0.75, 1.25, 1 + weightedAttributeDelta / 40)
```

技能经验：

```text
skillXpGainPerTick = jobSkillXpBase * jobMatchModifier * obedienceXpModifier * buildingXpModifier
skillModifier = 1 + skillLevelBonus(skillXp)
```

人口新增：

```text
population = countAlive(goblins)
freeHousing = max(0, maxPopulation - population)
regularPopulationSource = captive_bed_disposition_only
newGameCaptives = [{ type: "laborer", quality: "common", source: "开局" }]
```

新增人口：

```text
newGoblin = createGoblin({
  origin,
  inheritedTraitHint,
  attributeBias,
  startingSkillBias
})
goblins.push(newGoblin)
```

新增人口必须创建 `Goblin` 对象，并写入日志。住房和菌菇只限制承载、拥挤和口粮压力，不触发自动人口增长。死亡、逃亡和献祭不直接减少一个数字，而是把具体对象标记为非存活或移入历史记录。

俘虏苗床处置：

```text
click "培育新生"
require brainwashLevel > 0
set captive.disposition = "bed"
set captive.breedingState = "gestating"
set captive.gestationWeatherId = currentWeatherId
wait 1 calendar month
failureRisk = max(0.05, baseFailureRisk * weatherBreedingFailureModifier)
if randomRoll >= failureRisk:
    newGoblin = createGoblin(state, "captive_bed")
    apply weatherNewbornAttributePenalty
    apply attributeBias + floor(brainwashLevel / 25), clamped to 1-10
    apply startingSkill = skillBias * captiveQualityMultiplier
    push traitHint
    set brainwashLevel = round(brainwashLevel * 0.5)
else:
    set brainwashLevel = round(brainwashLevel * 0.1)
set captive.breedingState = "resting"
wait 1 calendar month
set captive.breedingState = "idle"
resting 只阻止再次执行 "培育新生"；"洗脑改造" 和 "做成食物" 仍可执行；休养结束后仍需 brainwashLevel > 0 才能再次培育
```

俘虏洗脑处置：

```text
click "洗脑改造"
require brainwashLevel < 100
require fungus >= 100 * weatherBrainwashCostModifier
spend fungus 100 * weatherBrainwashCostModifier
increase brainwashLevel by round(12 * qualityMultiplier * (1 + brainwashShedRatio) * weatherBrainwashGainModifier), clamped to 100
gain crudeKnowledge = 10 * qualityMultiplier * (1 + brainwashShedRatio)
show tooltip fields: brainwashLevel, brainwashCost, breedingSuccessRate, failureRisk, escapeRisk, resistanceRisk
require breedingState is "idle" or "resting"; blocked when breedingState is "gestating"; button disabled when brainwashLevel is already 100
```

俘虏自动洗脑：

```text
require technology "desire_enlightenment" is researched
click "自动洗脑：关/开" on a captive card
toggle captive.isAutoBrainwashEnabled for that captive only
each positive simulation tick:
  for each captive with isAutoBrainwashEnabled:
    require brainwashLevel < 100
    require breedingState is "idle" or "resting"
    require fungus >= 100 * weatherBrainwashCostModifier
    spend fungus 100 * weatherBrainwashCostModifier
    apply the same brainwashLevel and crudeKnowledge gain as "洗脑改造"
```

俘虏自动培育：

```text
require technology "public_nursery" is researched
click "自动培育：关/开" on a captive card
toggle captive.isAutoBreedEnabled for that captive only
each positive simulation tick:
  collect captives with isAutoBreedEnabled, brainwashLevel == 100, and breedingState is "idle"
  sort candidates by captive attribute value desc
  for each candidate:
    require fungus stock can cover at least 1 second of current ration consumption
    require free housing > 0
    apply the same gestating state as "培育新生"
```

俘虏战兽转化：

```text
require technology "human_beast" is researched
click "战兽转化" on a captive card
require breedingState is "idle" or "resting"; blocked when breedingState is "gestating"
remove captive from captives
append warbeast with speciesId = "converted_captive_beast"
set warbeast.name = captive.name
set warbeast.originalCaptiveId = captive.id
set warbeast.originalCaptiveType = captive.type
set warbeast.originalCaptiveRaceId = captive.raceId
set warbeast.isConvertedCaptive = true
show warbeast card race as original captive race name + "(兽)"
```

俘虏做成食物处置：

```text
click "做成食物"
require breedingState is "idle" or "resting"
gain fungus = 40 * captiveQualityMultiplier
remove captive from state.captives
```

菌菇消耗：

```text
fungusConsumerCount = alivePopulation + captiveCount
fungusConsumptionPerSecond = fungusConsumerCount * baseConsumptionPerSecond * (1 - foodReductionRatio)
if fungus is not enough to pay current ration:
    starvationSeconds += deltaSeconds
    every 3 game days:
        deaths = max(1, ceil((alivePopulation + captiveCount) * 0.1))
        randomly remove captives first and gain 10 fungus per dead captive
        if deaths remain, randomly mark deaths alive goblins as not alive
else:
    starvationSeconds = 0
```

断粮死亡必须随机作用到具体 `CaptiveState` 或存活 `Goblin` 对象。结算顺序优先抽取俘虏，俘虏从 `state.captives` 移除并按每名俘虏返还 10 菌菇；俘虏不足以覆盖本次死亡数时，剩余死亡数再随机作用到存活哥布林，哥布林死亡对象写入 `wounds += "starvation"`、清空 `jobId`。俘虏和哥布林死亡名单都必须写入重要日志。若当前没有俘虏且没有存活哥布林，则断粮计时不再造成额外死亡。

灭亡判定：

```text
if runMode is not "undecided" and alivePopulation <= 0 and captiveCount <= 0:
    clear current local save
    replace current state with createInitialState()
    show initial run mode selection
```

最后一个存活哥布林死亡且没有任何俘虏可继续繁衍，或最后一个俘虏被做成食物且没有存活哥布林时，当前局立即结束。结束后回到初始界面，重新显示正常模式/挑战模式选择；新初始状态仍按新局规则带 1 个普通质量村姑俘虏。

服从修正：

```text
obedienceModifier = clamp(0.75, 1.25, 1 + (obedience - 100) / 400)
accidentChance = baseAccidentChance * clamp(0.5, 2.0, 1 + (100 - obedience) / 100)
```

制作收益：

```text
craftedAmount = requestedAmount * (1 + craftRatio + tierCraftRatio + leaderCraftRatio)
```

## 7. 事件与日志

日志等级：

- 普通：建造、研究、交易成功。
- 重要：人口死亡、首次解锁、重置、稀有事件。
- 警告：食物不足、容量已满、服从过低、事故。

事件类型：

| 事件 | 条件 | 结果 |
| --- | --- | --- |
| 菌潮 | 已建立菌菇经济 | 菌菇产出临时 +50% |
| 长生苔 | 已建立菌菇经济 | 随机存活哥布林或俘虏 `eventLifespanYears +6` |
| 矿塌 | 矿井多且服从低 | 损失少量矿工或暂停矿井 |
| 熔炉爆燃 | 工业建筑多且稳固不足 | 损失煤渣/铁矿，日志警告 |
| 商队经过 | 外交已解锁 | 临时贸易倍率 |
| 酋长怒吼 | 服从过低 | 临时服从 +，粗识 - |
| 深渊低语 | 深渊门开启 | 获得深渊回响或触发代价 |
| 稳潮轮值 | 当前天气为稳潮 | 服从 +6，粗识 +4 |
| 湿涌菌簇 | 当前天气为湿涌 | 菌菇 +45，朽木 +15 |
| 湿涌漫坑 | 当前天气为湿涌且拥有浅矿井 | 碎石 -25，铁矿 -8 |
| 孢灯暴绽 | 当前天气为孢雨 | 菌菇 +70，粗识 +5 |
| 孢粉喷嚏 | 当前天气为孢雨 | 服从 -6，碎石 -12 |
| 风干柴堆 | 当前天气为穿洞风且拥有闷炭窑 | 朽木 +35，煤渣 +6 |
| 岩层回声 | 当前天气为穿洞风且拥有浅矿井 | 碎石 +20，粗识 +6 |
| 酸雾腐蚀 | 当前天气为酸雾且拥有粗熔炉 | 朽木 -20，铁矿 -8，铁片 -3 |
| 蚀痕壁画 | 当前天气为酸雾且拥有涂鸦墙 | 粗识 +12，碎石 +10 |
| 甜腥雾障 | 当前天气为欲风且拥有洗脑棚 | 服从 -8，粗识 +10 |

寿命随机事件直接写入个体对象的 `eventLifespanYears`，不作为全局资源或临时 buff。天气随机事件使用 `conditionId = current_weather`、`weatherIds`、可选 `requiredBuildingId` 和 `resourceChanges` 描述。所有天气随机事件共享季节上限：任一天气事件触发后写入 `statistics.lastWeatherEventSeasonKey`，同一季节内其他天气事件不再触发，换季后重新开放。`riskMode = neutral` 的天气机会事件不吃事故风险倍率；`riskMode = accident` 的天气事故继续受服从、稳固、契约和挑战风险修正影响。天气事件是一次性资源变化，不受当前天气产出倍率二次放大。

## 8. 完整开发范围

完整版本必须覆盖五个游戏阶段：地穴求生、矿坑氏族、地底城邦、黑铁帝国、深渊远征。开发时可以按里程碑推进，但策划范围不裁剪任何核心系统。

### 版本一：地穴求生

目标：完成基础生存闭环，让玩家能从零建立人口、食物、基础建筑和研究入口。

- 基础资源：菌菇、朽木、碎石、劳力、粗识、服从。
- 建筑：菌菇床、腐木圃、孢子沟、晾木架、滴水渠、潮痕桩、孢潮闸、窝棚、涂鸦墙、储物坑。
- 职业：捡柴工、采菌工、搬石工、涂鸦学徒。
- 科技：记号、朽木栽培、采菌、挖掘、潮痕观测、窝棚搭建、木架工艺。
- 系统：手动采集、资源上限、个体哥布林生成、菌菇消耗、职业分配、人口普查、日志、本地存档。
- 时间控制与存档控制：全局暂停/继续、顶栏重置存档、暂停状态保存、暂停时禁用主动状态变更、继续后不补算暂停收益。
- 个体需求：每个新哥布林拥有姓名、属性、特质、职业技能经验和稳定 ID；职业预设必须写回具体个体。

### 版本二：矿坑氏族

目标：完成采矿、加工、冶炼和早期风险管理。

- 新资源：煤渣、铁矿、铁片、皮革、骨片。
- 新建筑：浅矿井、碎石场、闷炭窑、工匠棚、粗熔炉、通风井、兽栏、劣酒桶。
- 新职业：矿工、熔炼工、抢掠兵、工匠。
- 新科技：采矿、冶金、闷炭术、通风支护、兽栏、大木棒、弓弩、粗制工具。
- 新工坊：木梁、石板、铁片、粗糙镐、加固背篓、锯齿斧、鼓风炉。
- 新事件：菌潮、矿塌、熔炉爆燃。
- 新俘虏：村姑、商队女账房、工坊女匠、采药女、苦修者；支持培育新生、洗脑改造、做成食物三类处置预览。
- 个体扩展：俘虏苗床繁衍生成带倾向属性和初始技能的哥布林，矿塌和熔炉爆燃可造成具体哥布林伤病或死亡。

### 版本三：地底城邦

目标：完成行政、贸易、掠夺、政策和祭祀的中期核心。

- 新资源：金币、钢锭、齿轮、账册、战利品、祖灵回响。
- 新建筑：酋长厅、黑市、训练坑、兵器坊、账房、祖灵祭坛、地底仓港。
- 新职业：账房、监工、巫医、工程师。
- 新科技：氏族规矩、计数、工程、货币、书写、祭祀、机械、钢铁。
- 外交对象：鼠人商队、灰矮人矿盟、蜥蜴人沼部、地精黑市。
- 掠夺目标：菌菇农洞、商队营地、矿盟哨站、地表村落。
- 俘虏系统：培育新生、洗脑改造、做成食物；所有收益和风险在执行前预览，俘虏按钮本身不再弹确认框。
- 政策系统：贸易/掠夺、恐吓/分配、深挖/稳固、祖灵/血月。
- 祭祀系统：祖灵回响、祖灵升级、战利品献祭。
- 自动化：工程师自动制作、职业预设、隐藏已完成研究。
- 个体扩展：领袖任命、领袖晋升、固定职业、按职业/特质/伤病筛选人口普查。

### 版本四：黑铁帝国

目标：完成高阶工业、符文科技、军工扩张和后期资源回流。

- 新资源：焦油、黑铁、符文板、魔晶、威望。
- 新建筑：焦油井、深炉、黑铁要塞、黑铁居所、符文机房、战争营地。
- 新职业：符文匠、战争头目、深矿师。
- 新科技：地表学、外交术、符文学、黑铁铸造、帝国法典。
- 新工坊：钢锭、齿轮、黑铁、符文板、战旗、符文刻刀、深炉阀门。

## 9. 数值校验目标

完整节奏建议：

- 0-1 分钟：完成第一个菌菇床和窝棚。
- 1-5 分钟：解锁研究，建立采菌/捡柴职业循环。
- 5-15 分钟：解锁储物坑、浅矿井、工匠棚。
- 15-30 分钟：完成第一条加工链，建造粗熔炉。
- 30-60 分钟：进入矿坑氏族阶段，准备军工和外交系统。
- 1-3 小时：建立酋长厅、黑市、训练坑和祖灵祭坛，完成城邦核心。
- 3-8 小时：进入钢铁、机械、政策分支和自动制作阶段。
- 8-20 小时：建成黑铁要塞、符文机房和高阶外交网络。
- 20-60 小时：打开深渊门，完成深渊契约和第一次帝国迁徙。
- 多周目标：完成挑战模式、威望天赋树和多路线数值构筑。

数值风险：

- 菌菇消耗过高会导致早期惩罚感过强。
- 容量过低会造成频繁卡死，需要保证仓储解锁及时。
- 手工制作若成本过高，会让玩家只等待而无决策。
- 苗床繁育过快会让职业选择失去意义，应受俘虏获取、住房和菌菇消耗共同限制。
- 个体对象过多会造成 UI 和存档压力，应提供筛选、分页/优先列表、压缩保存和迁徙清理规则。
- 个体属性加成过高会破坏增量数值，应把属性修正控制在温和区间，主要让技能和长期培养提供差异。

## 10. 当前实现同步基线

本节同步当前 `game/` 静态应用的真实落地状态。若前文 TypeScript 草案或版本里程碑与本节冲突，以本节为准。

### 10.1 运行入口和模块顺序

当前应用不使用构建工具，直接打开 `game/index.html` 运行。脚本加载顺序是运行契约、ID、文本、定价、定义、日期系统、天气系统、状态、存档、各系统、渲染、事件、主入口。主入口 `game/scripts/main.js` 会先加载存档，失败时创建新状态，然后同步俘虏资源、应用解锁和建筑效果，最后启动定时循环。顶栏重置按钮由 `game/ui/events.js` 二次确认后调用 `game/state/save.js` 清除本地存档，再用 `game/state/initialState.js` 创建新状态并重新渲染。新状态的 `challenges.runMode` 默认为 `undecided`，首屏地穴正文只显示“地穴控制台”和“新局模式”两个卡片；“新局模式”卡片内同时提供正常模式入口和挑战模式入口。玩家选择正常模式后写入 `normal`，选择挑战后写入 `challenge` 与活动挑战 ID；开局后地穴正文不再显示“新局模式”卡片。

当前顶层系统模块包括：日期、天气、资源、建筑、科技、职业、人口、制作、外交、掠夺、俘虏、政策、祭祀、契约、远征、威望、挑战、事件、生产、资源流量分析和模拟。UI 模块包括 `render.js` 与 `events.js`。

### 10.2 当前状态和存档字段

当前存档版本为 24。`GameState` 真实字段包括：

- `version`：存档版本整数。
- `isPaused`：暂停状态；暂停时主循环不推进，主动状态变更被禁用或拦截。
- `lastActiveTimestamp`：最后一次允许模拟推进的 Unix 毫秒时间戳。
- `resourcesById`：资源状态字典，每项包含 `id`、`value`、`maxValue`、`isVisible`、`perSecond`。
- `buildingsById`：建筑状态字典，每项包含 `id`、`owned`、`active`、`isUnlocked`。
- `technologiesById`：科技状态字典，每项包含 `id`、`isUnlocked`、`isResearched`。
- `jobsUnlockedById`、`tabsUnlockedById`、`upgradesUnlockedById`、`craftsUnlockedById`、`policiesUnlockedById`：各类解锁字典。
- `goblins`、`leaderGoblinId`：哥布林对象数组和领袖引用；人口数量从存活对象派生。
- `policies`：政策选择字典，key 为政策组 ID，value 为政策 ID。
- `pacts`：深渊契约启用字典，key 为契约 ID。
- `activeExpedition`：当前远征状态或 `null`。
- `activeDiplomacyMissions`：当前返程中的贸易队和掠夺队数组；每项保存行动类型、地点、势力、出战成员、剩余秒数、总秒数和出发时冻结的结算数值。
- `challenges`：包含 `runMode`、`activeChallengeId` 和 `completedById`；`runMode` 可为 `undecided`、`normal` 或 `challenge`。
- `calendar`：日期运行时状态，包含 `elapsedDays`、`dayProgressSeconds`、`isCalendarUnlocked` 和 `calendarEpochDay`。
- `weather`：天气运行时状态，包含 `currentWeatherId`、`startedElapsedDay`、`nextChangeElapsedDay` 和 `randomSeed`。
- `captives`：俘虏运行时数组；新局默认包含 1 个普通质量村姑俘虏；每个俘虏保存 `brainwashLevel`、`isAutoBrainwashEnabled`、`isAutoBreedEnabled`、`breedingState`、`gestationWeatherId`、`gestationSecondsRemaining` 和 `restSecondsRemaining`。
- `prestige`：包含 `legacy` 和已购买 `perks`。
- `statistics`：累计统计、关系、自动制作选择、成就标记和事件冷却等运行数据。
- `activeTabId`、`logs`：界面标签和日志。

`SaveData` 只保存进度，不保存静态定义副本。导入存档必须经过版本迁移和字段补齐；加载后会重新按定义计算容量、解锁、建筑效果、寿命科技加成和俘虏资源。

月末自动存档由 `game/systems/simulation.js` 在未暂停主循环中触发：`game/systems/calendar.js` 按 30 个游戏日计算完整月份序号，当本 tick 后的月份序号大于 tick 前月份序号时，调用 `game/state/save.js` 的 `saveToLocalStorage` 写入与手动保存相同的本地存档。自动存档不改变 `SaveData` 字段，不需要额外迁移；保存成功会写入普通日志，浏览器本地存储异常会写入警告日志。

`resourcesById` 当前包含恶名 `infamy` 和善名 `goodwill`。旧存档升到 v15 时不复制静态定义，恢复流程会从当前资源定义补齐这两个资源的默认状态；交易成功增加善名，高阶贸易势力读取善名门槛，掠夺成功增加恶名并减少善名，掠夺失败减少恶名，高阶掠夺目标读取恶名门槛。

v3 存档迁移会从旧存档推断 `challenges.runMode`：有活动挑战则为 `challenge`，已有普通资源、建筑、科技或人口进度且无活动挑战则为 `normal`，完全空的新局则为 `undecided`。v4 存档迁移会补齐 `calendar`，旧存档从春第 1 日、未解锁历法开始。v5 存档迁移会移除旧自然增长进度；完全空的新局若没有俘虏，会补 1 个普通质量村姑俘虏。v6 存档迁移会给旧俘虏补齐洗脑程度、孕育状态和休养倒计时。v7 存档迁移会把旧 10 秒 1 天口径下的 `calendar.dayProgressSeconds`、`gestationSecondsRemaining` 和 `restSecondsRemaining` 按 10:1 缩放到 1 秒 1 天口径。v10 存档迁移会补齐 `weather`，旧存档从稳潮开始并在后续游戏日进入天气轮换。v11 不改写存档数组，加载时由当前定义补齐潮痕观测、通风支护和天气调控建筑状态。v12 会给天气状态补齐 `randomSeed`，让不同新局拥有不同天气序列，同时保持同一存档读档稳定。v16 会补齐 `activeDiplomacyMissions` 空数组，旧存档默认没有返程中的贸易队或掠夺队。v17 会给俘虏补齐 `isAutoBrainwashEnabled`，v18 会给俘虏补齐 `isAutoBreedEnabled`，v19 会给哥布林和俘虏补齐年龄、基础寿命、科研寿命、事件寿命和老死检查计数。v20 会把旧 `*LifespanMonths` 字段迁移为 `*LifespanYears` 字段，旧年龄月数折算为年；旧俘虏若没有有效年龄，会按类型补基础年龄。v23 会补齐 `warbeasts` 空数组，旧存档默认没有已捕获战兽。v24 会给战兽规范化俘虏转化来源字段，支持 `human_beast` 科技生成的转化战兽显示原种族 `(兽)`；已完成 `public_nursery` 的旧存档会补齐 `human_beast` 为可研究状态。v25 会把旧俘虏种族 ID `mire_human`、`frontier_human`、`hill_dwarf`、`deep_dwarf`、`wood_elf`、`moon_elf` 分别合并为 `human`、`dwarf`、`elf`，战兽转化来源字段同步规范化。v26 会给哥布林和俘虏补齐 `faithId`：旧档中已有祖灵祭坛的哥布林会获得 `goblin_ancestor`，未建祭坛的哥布林保持 `null`；旧俘虏按当前种族主信仰规则随机补齐或保持无信仰；战兽仍不保存信仰字段。

### 10.3 当前日期与日志

日期由 `game/systems/calendar.js` 统一推进，主循环未暂停时现实每步进 1 秒，游戏时间步进 1 天。未研究 `calendar`（历法）前，时间只按春、夏、秋、冬循环，每季 90 天，全年 360 天；日志前缀格式为 `春-第n日-xxxxx`，状态栏显示同一季节日。

研究 `calendar` 后，系统把研究完成当时的完整游戏日写入 `calendar.calendarEpochDay`，并将当天作为哥布林历第 1 年的起点。之后日志前缀格式为 `哥布林历xx年x月x日(春) - xxxxx`；哥布林历每年 12 个月、每月 30 天，年份从 1 开始，括号内季节继续沿用地穴自然季节循环。历法研究完成日志本身也使用哥布林历格式。

天气由 `game/systems/weather.js` 在日期推进后检查。新局天气为 `clear`（稳潮），并生成一个保存到 `weather.randomSeed` 的本局天气随机种子；首段天气和后续每段天气都会按本局随机种子在 30-90 个游戏日之间抽取持续时间，再进入按季节权重的确定性轮换。天气状态保存当前天气 ID、开始日、下次变化日和本局随机种子；暂停时天气不推进，读档后继续当前天气段，刷新页面不会重新抽取；不同新局因为随机种子不同，应产生不同天气序列。当前天气显示在左栏状态和右侧日志日期旁；天气变化会写入普通日志。俘虏孕育会在开始时把天气 ID 写入 `captive.gestationWeatherId`，本次孕育的成功率和新生属性惩罚按开始孕育时天气结算。

当前天气定义：`clear` 稳潮无修正；`damp` 湿涌使菌菇 +15%、朽木 +10%、矿道 -5%；`spore_rain` 孢雨使菌菇 +30%、矿道 -10%；`cave_wind` 穿洞风使朽木 -5%、矿道 +12%；`acid_fog` 酸雾使菌菇 -20%、朽木 -10%、矿道 -8%、炉火 -5%；`lust_wind` 欲风使洗脑消耗 -50%、洗脑提升 +50%、孕育失败率 -50%、本次新生全属性 -1。天气只影响持续产出、资源流量预估和明确列出的俘虏处置修正，不影响手动采集、贸易、掠夺、远征结算和一次性事件奖励。潮痕桩和通风井提供 `weatherNegativeMitigationRatio`，只削弱资源产出的负面天气，累计上限为 85%；孢潮闸提供 `weatherPositiveAmplificationRatio`，只放大资源产出的正面天气。两类天气建筑不改变欲风的洗脑和孕育修正。

天气随机事件已经作为普通事件系统的一部分落地：事件轮询仍为每 5 秒一次，当前天气匹配 `weatherIds`、额外建筑条件满足且本季尚未触发过天气事件时才会进入概率判定。当前天气事件共有 10 个：稳潮轮值、湿涌菌簇、湿涌漫坑、孢灯暴绽、孢粉喷嚏、风干柴堆、岩层回声、酸雾腐蚀、蚀痕壁画和甜腥雾障；它们只做一次性资源增减和日志反馈，不改变天气段长度，也不改变天气生产倍率。当前非天气寿命事件包含长生苔：已建立菌菇经济后低频触发，随机选择存活哥布林或俘虏，写入 `eventLifespanYears +6`。

### 10.4 当前暂停门控

暂停不是资源倍率归零，而是主循环时间门控。`main.js` 在 `state.isPaused === true` 时不调用模拟推进，并保持 `lastActiveTimestamp` 为暂停前的权威时间。暂停状态会写入存档，重新打开后仍保持暂停，不补算暂停期间收益。

暂停时允许：查看界面、切换标签、日志过滤、保存、加载、导出、导入、重置存档。暂停时禁止或拦截：手动采集、购买建筑、研究科技、制作配方、自动制作切换、职业分配、任命领袖、贸易、掠夺、俘虏处置、政策切换、献祭、契约切换、远征、威望迁徙和天赋购买。

### 10.5 当前公式口径

人口：

```text
alivePopulation = count(goblins where isAlive)
housingMax = sum(ownedBuilding * effects.housingMax)
freeHousing = max(0, housingMax - alivePopulation)
rawBuildingLaborUsage = sum(activeProductionBuilding * effects.laborUsage)
laborUsageReductionRatio = min(0.75, sum(ownedBuilding * effects.laborUsageReductionRatio))
buildingLaborUsage = rawBuildingLaborUsage * (1 - laborUsageReductionRatio)
labor = max(0, alivePopulation * 10 - buildingLaborUsage)
isProductionLaborOverloaded = buildingLaborUsage > alivePopulation * 10
```

当前人口常量：单个哥布林或单个俘虏菌菇消耗 `3.125/s`；战兽按物种 `foodConsumptionRatio` 折算口粮口数，休养时口粮翻倍；战兽屠宰或断粮回收固定获得 100 菌菇。断粮死亡检查间隔为 3 个游戏日；每次断粮死亡比例为当前口粮口数的 10%，向上取整且最少 1 个；断粮死亡优先移除俘虏，每名断粮死亡俘虏返还 10 菌菇，其次移除战兽并按屠宰收益回收，最后才杀死哥布林。哥布林基础寿命为 72 年，成长寿命上限为 36 年；俘虏基础寿命按质量随机，普通 30-55 年、熟练 45-70 年、精英 60-85 年、传奇 80-100 年；俘虏基础年龄按类型随机，例如村姑 12-18 岁；老死骰首次概率为 10%，每月递增 10%；长生苔事件单次寿命加成为 6 年。无加成下 5 个菌菇床（每个 `0.125/tick`，5 tick/秒）正好覆盖 1 个口粮口数。住房只决定承载和拥挤，菌菇只决定存活与关押/饲养消耗，不会推进自然增长。常规新增哥布林可以通过俘虏卡牌的“培育新生”生成 `origin = captive_bed`，也可以通过已驯化战兽的“作为苗床”生成 `origin = warbeast_bed`。俘虏洗脑程度为 0 时不可培育；每次培育需要 1 个月孕育，按洗脑程度判定失败概率，成功后洗脑程度降为结算前的 50%，失败后洗脑程度降为结算前的 10%，成功或失败后都需要 1 个月休养；休养完成后同一俘虏仍需洗脑程度大于 0 才能再次培育。完成人即是兽后，俘虏卡牌可执行“战兽转化”，该操作移除俘虏并生成 `converted_captive_beast` 战兽，战兽保留俘虏姓名，种族栏显示原俘虏种族加 `(兽)`。战兽每次驯化增加 25 点进度，100 点后可作为苗床；战兽苗床不需要洗脑，孕育 1 个月后必定生成新哥布林并进入 1 个月休养。完成公用苗床后，自动培育要求俘虏洗脑 100%、空闲、菌菇库存至少覆盖一秒现有口粮且住房有空位；多个自动培育俘虏同时满足条件时，按“俘虏类型属性偏置总和 * 质量倍率”从高到低启动。新哥布林由 `createGoblin` 生成稳定 ID、确定性姓名、六项属性、四类初始特质之一和十项技能键，再写入寿命拆分字段，并应用俘虏或战兽来源偏置。

建筑生产：

```text
if isProductionLaborOverloaded and buildingId != fungus_bed: buildingProduction = 0
perTickBuildingOutput = effectPerTick * TICKS_PER_SECOND * activeBuildingCount
perSecondBuildingOutput = effectPerSecond * activeBuildingCount
productionBuildingLaborUsage = effects.laborUsage * activeBuildingCount * (1 - laborUsageReductionRatio)
crudeFurnace: 消耗朽木和碎石，产铁矿和铁片
charcoalKiln: 消耗朽木，产煤渣
deepFurnace: 消耗焦油，产钢锭和黑铁，深炉阀门最多降低 75% 焦油消耗
runeMachine: 消耗魔晶，产粗识
```

除菌菇床外，当前所有持续产出、资源转换或产出倍率类生产建筑都定义 `laborUsage`。该字段表示启用建筑占用的劳力，不作为一次性建造成本保存；人口系统每次刷新劳力时统一扣除，占用过高时可用劳力最低为 0。当减免后的建筑总占用严格大于存活哥布林提供的 `alivePopulation * 10` 时进入生产劳力过载：菌菇床继续产出，其他建筑的持续产出、资源转换、建筑提供的产出倍率和天气调控效果全部停用；职业、口粮消耗和契约代价不属于建筑生产，仍按原规则推进。绞盘廊和监工台通过 `laborUsageReductionRatio` 降低总占用，减免总比例上限为 75%。劳力资源卡片浮窗要把该公式拆成玩家可读明细：`alivePopulation * 10` 的人口派生来源、每类建筑 `effects.laborUsage * activeBuildingCount` 的占用、全局减免比例，以及减免后实际占用。
