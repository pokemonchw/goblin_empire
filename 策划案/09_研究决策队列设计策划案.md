# 研究决策队列设计策划案

> 本文件是研究工作台“决策队列”的专项产品、算法与实现规格。科技价格、效果和解锁关系以 `04_科技外交信仰与重置.md`、`game/data/definitions.js` 为准；页面结构、焦点与刷新边界以 `07_完整交互设计策划案.md` 为准。本文件只决定“哪些研究值得现在关注、如何排序、怎样解释”，不改变研究资格、价格、效果或图鉴顺序。

## 1. 设计结论

研究决策队列是一个只读的研究参谋，不是把可见科技按层级截取七项，也不是自动研究队列。它必须同时回答四个问题：

1. 帝国当前最需要哪种知识能力；
2. 哪项科技能直接推进系统、解除经营瓶颈或铺平依赖路径；
3. 这项科技为何现在比其他科技重要；
4. 玩家下一步应积累什么、完成哪个前置或提升哪种容量。

队列仍使用最多七项的固定骨架，但三个区段具有严格不同的语义：

| 区段 | 上限 | 回答的问题 | 硬性条件 |
| --- | ---: | --- | --- |
| 当前目标 | 1 | “现在应围绕哪项研究组织知识生产？” | 正式揭示、前置完成、尚缺资源、容量足够且所有缺口有现存取得路径 |
| 可立即研究 | 3 | “手中知识现在花在哪里最有效？” | 正式揭示、前置和价格均满足；优先覆盖不同意图与路线 |
| 先处理 | 3 | “什么阻止关键研究继续推进？” | 容量硬阻断、无来源资源、近端关键前置或数据配置异常；必须给出唯一处理方向 |

同一科技只能进入一个区段。已完成科技保留在图鉴但不占队列；`unknown` 永不出现；`preview` 只用于解释具名依赖，默认不占正式槽位。研究操作仍只存在于固定检查器，队列不得自动扣费、自动改变职业或自动启停建筑。

## 2. 旧设计为何无效

原方案只按“里程碑、直接解锁数量、层级、路线顺序”排列，存在以下根本问题：

- 它不知道粗识、账册和魔晶的库存、容量、净流量与来源，因此会把永远攒不到的科技当作当前目标。
- 它不知道“缺资源”和“缺前置”是两类完全不同的行动问题，也不能指出主导缺口。
- 它把所有数组型解锁等价计数，无法区分主系统入口、生产能力、内容开放和普通强化。
- 它只对“可立即研究”做路线去重，不能避免三个选项虽然路线不同却解决同一问题。
- 它不计算依赖距离。遥远高价值节点会压过只差一步、能立即解除当前瓶颈的铺路科技。
- 它使用渲染函数临时排序，没有独立数据契约，无法单元测试、复用或稳定解释。
- 推荐签只复用图鉴文案，没有展示“推荐理由 + 唯一瓶颈”，玩家仍需自己重读全部成本和依赖。
- 预览科技可能被当作目标，造成未知内容泄露和没有行动入口的伪建议。

## 3. 设计目标与非目标

### 3.1 目标

- **可行动**：每项都能导向“立即研究、积累资源、主动取得、扩容、补前置”之一。
- **懂经济**：读取与模拟一致的库存、容量和每秒净流量，不伪造资源来源。
- **懂依赖**：识别 AND/OR 前置、最近未完成依赖和循环/缺失配置。
- **懂收益**：系统入口和新能力优先于同层普通数值强化；生存与停摆问题可压过远期里程碑。
- **保留选择**：三个可研究项优先覆盖不同经营意图，其次覆盖不同路线。
- **可解释**：排序字段都能还原为玩家可验证的理由，不向玩家展示神秘总分。
- **确定性**：相同状态得到相同结果；不使用随机数、DOM 顺序或本地时间。
- **稳定交互**：自动 tick 不重建搜索框、组合输入、按压目标、打开浮窗或选择所依赖的目录节点。
- **可测试**：候选生成与 UI 分离，固定状态夹具能验证排序、分段、多样性和不泄露不变量。

### 3.2 非目标

- 不做自动研究 AI，不替玩家支付资源。
- 不穷举整棵科技树的全局最优路线，不承诺数小时后的精确收益。
- 不从中文描述猜测效果；智能判断只读取结构化标签、解锁包、价格和依赖。
- 不修改科技价格、前置、效果、揭示规则或六路线图鉴排序。
- 不用单一浮点总分掩盖设计取舍，也不向玩家展示内部档位。

## 4. 权威输入与派生契约

### 4.1 权威输入

- `GameState.technologiesById`：是否揭示、解锁与完成。
- `GameState.resourcesById`：库存、容量与当前每秒净流量。
- `TechnologyDefinition.price`：真实研究价格。
- `prerequisiteTechnologyIds`：必须全部完成的 AND 前置。
- `alternativePrerequisiteTechnologyIds`：至少完成一项的 OR 前置。
- `lineId`、`tier`、`layoutOrder`：稳定研究位置，只作依赖邻近和最终平局。
- `nodeSize`、`effectTags`、`unlocks`、`recommendedFor`：推进价值、经营意图和公开解释。
- 已开放的手动采集、制作、外交、祭祀和远征入口：用于判断离散资源来源。未接入某类来源时必须保守标记，不能假定存在。

## 5. 知识经济诊断

### 5.1 三类研究货币

- **粗识**：早期基础研究货币，允许持续生产与明确的主动取得路径。
- **账册**：治理和中期技术货币。没有正净流量且未开放离散取得方式时，视为无来源，不计算等待时间。
- **魔晶**：后期符文与深渊技术货币。只有真实生产、制作、贸易或远征入口已开放时才算有来源。

对每个价格项依次判断：库存是否足够、需求是否超过容量、净流量是否为正、是否存在已开放的主动/离散来源。不得仅凭资源已经显示就判定“可取得”。

### 5.2 等待时间

只对正持续净流量计算等待时间：

```text
单资源等待秒数 = max(0, 需求量 - 当前库存) / 当前净流量
多资源等待秒数 = 各缺口等待秒数的最大值
```

多种资源并行积累，因此取最大值而不是求和。存在手动或离散来源时显示取得方式，不显示伪精确倒计时。任一缺口无来源或价格超过容量时，总等待为不可计算，科技不能进入“当前目标”。暂停时保留最近净流量诊断，但文案明确等待不推进。

等待档位用于排序，不直接作为价值：`now` 为 0，`short` 为 0-60 秒，`medium` 为 60-300 秒，`long` 为 300 秒以上，`unreachable` 为不可达。短等待不能无条件压过关键系统入口。

### 5.3 主导缺口

多资源价格只显示一个主瓶颈。主导缺口按 `缺少量 / 需求量` 降序选择；若比例相同，再按价格数组稳定顺序。检查器继续显示全部成本，队列签只承担快速决策，避免三行重复数字淹没理由。

### 5.4 容量与来源硬阻断

- 价格量大于当前容量上限时为容量硬阻断，紧迫档位为 4。
- 容量足够但库存不足且没有任何取得路径时为来源硬阻断，紧迫档位为 3。
- 有手动/制作/贸易/远征路径但无持续流量时为离散来源，不进入自动等待目标，可进入“先处理”并给出真实入口。
- “资源尚未显示”不等于“资源不存在”；必须查询稳定 ID 和已开放系统。
- 找不到任何路径时写“当前未发现可用取得路径”，不得模糊写成“再等等”。

### 5.5 建造需求反推研究解法

研究参谋必须读取当前已开放建筑的权威下一座价格。只有“资源库存已到容量上限、建筑当前已解锁、下一座价格高于容量上限”同时成立，才建立建造容量阻断；未揭示建筑、未满仓资源和远期推测需求不得触发。

系统从正式揭示且未完成的科技中沿结构化 `unlocks.buildings` 查找直接解锁项，并验证被解锁建筑的 `effects` 确实提高受阻资源容量。推论链固定为：

```text
已满资源 + 当前受阻建筑 → 提高该资源容量的建筑 → 直接解锁该建筑的科技
```

命中项获得最高紧迫档和独立压力匹配资格，优先于只因解锁项目较多而靠前的阶段或军备科技。理由必须具名说明扩容建筑、资源和受阻建筑，例如：“解锁‘储物坑’扩充菌菇容量，使‘菌菇床’的下一次建造可达。”算法不得为某个科技 ID 写死特殊排序。

### 5.6 资源保留与机会成本

研究参谋不能把每项科技孤立评价。生成“当前目标”后，该科技成为本轮 `strategicTargetId`；所有可立即研究项都要检查其价格是否会消耗当前目标仍缺少的同种资源。

```text
主冲突量 = min(立即研究项价格量, 当前目标同资源缺口)
冲突比例 = 主冲突量 / 当前目标同资源缺口
自动补回秒数 = 主冲突量 / 该资源正净流量
```

- 多种资源冲突时只选择冲突比例最高的一种作为签面主提示，检查器仍显示完整价格。
- 冲突比例达到 50% 为强冲突，低于 50% 为弱冲突；同一战略与危机档内，无冲突项优先于强冲突项。
- 有正净流量时显示“约延迟 mm:ss”；没有正净流量时显示“当前无法自动补回”，不得把手动或离散来源换算为伪精确时间。
- 能解决严重生存、治理或停摆压力的科技不受资源保留降序；危机处置高于远期目标。
- 资源冲突不是禁止研究。玩家仍可在检查器中选择并研究，参谋只解释代价，不替玩家锁按钮。
- 当前没有可达等待目标时不建立资源保留关系，避免凭空指定战略。

## 6. 依赖图诊断

### 6.1 AND 与 OR 前置

- `prerequisiteTechnologyIds` 中每项都必须完成。
- `alternativePrerequisiteTechnologyIds` 为空时视为满足；非空时至少完成一项。
- AND 缺口的行动方向按定义顺序选择首个近端未完成项。
- OR 缺口选择依赖距离最近、当前可达且推进价值最高的一项；不得要求玩家完成全部任选前置。

### 6.2 依赖距离

依赖距离是从候选到最近可行动未完成前置的步数：前置已满足为 0，直接缺一个为 1，以此类推。比较战略价值接近的科技时，距离短者优先。递归必须维护访问集合：发现循环、缺失 ID 或深度异常时返回 99，在开发环境记录配置错误，并将候选降级到“先处理”，不能死循环或假装可达。

### 6.3 铺路研究

当高价值科技受前置阻断时，参谋应推荐最近的可行动铺路节点，而不是反复展示遥远目标。回溯规则：

1. 最大解释深度为三层；
2. 优先可立即研究的前置；
3. 其次选择有持续来源且等待不超过 300 秒的前置；
4. 再比较推进档、意图匹配和路线位置；
5. 铺路项文案必须写明“为‘目标科技’铺路”，不能伪装成它自身的直接收益。

## 7. 经营意图与收益判断

意图使用封闭 ID，禁止解析中文描述临时猜测：

| 意图 ID | 含义 | 结构化来源示例 |
| --- | --- | --- |
| `survive_food` | 解决口粮与人口生存 | 食物产出、消耗降低、坏事件移除 |
| `increase_output` | 提升资源取得 | 增产标签、生产倍率、主动采集强化 |
| `raise_capacity` | 提升研究或关键资源容量 | 容量标签、容量解锁 |
| `unlock_system` | 开放标签、职业、建筑、配方或规则 | 非空解锁包、系统标签 |
| `automation` | 减少重复操作 | 自动制作、自动分配、批量规则 |
| `governance` | 提升治理、服从与账册经济 | 治理标签、政策/人口管理解锁 |
| `military_readiness` | 提升军力、掠夺和防御 | 军事标签、兵种/战争解锁 |
| `faith_growth` | 推进献祭与信仰 | 信仰标签、祭祀解锁 |
| `abyss_growth` | 推进远征与深渊循环 | 深渊标签、远征/裂隙解锁 |
| `general_growth` | 无法进一步分类的常规强化 | 仅作低优先级兜底 |

静态定义后续可增加 `decisionHints`，但必须使用受控字段，例如 `intentIds`、`pressureResourceIds`、`stageIds`、`enablesAcquisitionResourceIds`。不得加入无键契约的 `meta` 或自由文本评分。

## 8. 分档评价，不使用伪精确总分

### 8.1 推进档位

| 档位 | 定义 |
| ---: | --- |
| 4 | 里程碑且直接开放主系统、阶段入口或关键循环 |
| 3 | 开放至少两个可行动内容，或开放一种此前不存在的关键资源/能力 |
| 2 | 开放一个内容、完成关键铺路，或自身为里程碑 |
| 1 | 与当前最高进度相邻的明确强化 |
| 0 | 遥远、重复或当前无法识别作用的普通强化 |

解锁数量只是输入，不能单独代表价值。开放一个全新主标签通常高于开放三个显示项；这类语义必须由结构化解锁类别或 `decisionHints` 表达。

### 8.2 紧迫档位

| 档位 | 定义 |
| ---: | --- |
| 4 | 当前关键科技存在容量硬阻断，或研究能力完全停摆 |
| 3 | 关键研究货币无来源、经营危机存在明确研究解法 |
| 2 | 当前层有可立即执行的直接解法，或资源持续爆仓/净亏损 |
| 1 | 一至两步内的关键铺路节点 |
| 0 | 当前没有经营或依赖紧迫性 |

### 8.3 确定性比较键

候选按下列字段逐项比较：

```text
[紧迫档位降序,
 当前经营意图匹配降序,
 资源冲突档位升序,
 推进档位降序,
 依赖距离升序,
 等待档位升序,
 价格已备齐比例降序,
 未覆盖意图优先,
 未覆盖路线优先,
 路线定义顺序升序,
 tier 升序,
 layoutOrder 升序]
```

多样性只在有限槽位选择阶段应用，不改变科技档案的基础战略顺序。界面不显示内部档位或总分，只显示可以验证的理由和瓶颈。

## 9. 三个区段的生成算法

### 9.1 公共候选池

1. 遍历静态科技定义并读取运行状态。
2. 排除 `unknown` 和 `researched`。
3. 为其余节点建立 `ResearchDecisionProfile`。
4. `preview` 只参与依赖回溯和匿名远景，不能进入目标或可立即研究。
5. 先生成当前目标，再生成可立即研究，最后生成先处理；分配后登记稳定 ID，禁止重复。

### 9.2 当前目标

候选必须同时满足：正式揭示、未完成、前置已完成、当前资源不足、价格不超过容量、所有缺口存在来源。排序先看紧迫和推进，再看等待档；同档且处于同一等待区间时，优先完整价格已备齐比例更高者。长等待普通强化不应占据目标；长等待里程碑可以保留。入选项同时成为本轮战略目标，供可立即研究项计算资源机会成本。

文案结构固定为：

```text
推荐理由。唯一瓶颈与行动方向。
```

示例：“完成后会开放工坊与配方。还缺 18 粗识，按当前流量约 01:12。”

### 9.3 可立即研究

从 `available` 候选中最多选择三项：

1. 第一项取基础比较键最高项；
2. 第二项优先不同 `primaryIntentId`，再优先不同路线；
3. 第三项继续覆盖未出现意图和路线；
4. 若某项能直接解决 `critical` 经营压力，可以打破多样性并插队；
5. 没有足够不同意图时按基础顺序补足，不为凑满而加入低价值未知项。
6. 除严重危机的直接解法外，会挪用当前战略目标缺口资源的候选降序，并显示补回延迟；它仍可被玩家选择和研究。

理想的三个选项分别表达“解除当前危机、推进主系统、强化另一条路线”，而不是三个名称不同的增产节点。

### 9.4 先处理

优先级固定为：

1. 研究经济停摆或容量硬阻断；
2. 高推进科技的无来源价格资源；
3. 当前目标的一步关键前置；
4. 为近端里程碑铺路的可行动节点；
5. 依赖循环、缺失定义等配置异常；
6. 队列完全为空时最多一个匿名预览远景。

每项只能显示一个主瓶颈，但检查器保留完整成本和前置。示例：

- “账册容量不足：需要 120，当前上限 80；先提升账册容量。”
- “魔晶尚无可用取得路径；先开放符文生产或深渊取得方式。”
- “先完成‘氏族规矩’，它是当前最近的必需前置。”
- “为‘迁徙法典’铺路；当前可先研究‘帝国史’。”

## 10. 解释文案契约

每个队列项必须有两段受控信息：

- `reasonText`：为何值得研究，来自推进档、经营意图和结构化解锁。
- `bottleneckText`：为何现在不能研究或如何行动，来自真实价格、流量、容量和前置。

禁止文案：

- “系统推荐”“综合评分最高”“性价比最好”：无法验证。
- “很快就能完成”：没有持续流量计算。
- “缺少资源”：没有指出资源、缺口或路径。
- “需要前置”：没有指出最近具名前置。
- 直接输出英文稳定 ID：必须通过定义表取得中文名；开发日志除外。

预览态只允许用途剪影和已经揭示的具名前置，不公开正式名称、价格、效果数值、后继或内部评分。

## 11. 稳定性与刷新策略

### 11.1 重新派生事件

以下事件允许立即重新生成队列：研究完成、科技揭示、价格资源容量变化、取得路径开放/关闭、知识净流量从正变非正或反向、关键经营压力跨档。普通 tick 只刷新缺口和等待文案；成员未失效时不应因小数流量波动频繁换位。

建议运行时维护不入存档的 `ResearchDecisionRuntime`：

```text
stableSnapshot
pendingSnapshot
pendingSinceTimestamp
structureSignature
```

结构事件立即替换；普通排序变化连续保持 3 秒才替换；新候选高出至少一个紧迫或推进档可立即插队。暂停时成员保持，只把操作结论改成“已暂停”，等待时间停止倒数。

### 11.2 DOM 刷新边界

- 搜索输入是常驻原节点，搜索、筛选和队列刷新只替换 `.research-command-body`。
- 输入法组合期间不得刷新结果；组合结束后合并刷新。
- 选择研究签只更新旧/新签 `aria-current`、相关浮窗和 `.research-inspector`。
- 自动 tick 不得替换聚焦输入、textarea、select、组合目标、指针按压目标或打开的交互浮窗。
- 不得通过重建输入后恢复焦点/光标来伪装节点保持。
- 队列成员变化不得清空仍已揭示的检查器选择；科技完成后保持选择并显示已完成状态。

## 12. UI 结构与无障碍

- 队列签和图鉴签使用同一共享视觉语言、密度、状态字形、边框、焦点和禁用态。
- 队列签额外显示一行“推荐理由 + 唯一瓶颈”，不新增行内研究按钮。
- 点击、Enter 或 Space 只选择；唯一研究按钮在固定检查器。
- 当前签使用 `aria-current="true"`；三个区段标题包含可见数量。
- 状态不得仅靠颜色表达，必须同时有文字、字形和边框差异。
- 窄屏按 `07` 规定切为单栏/抽屉，不能依赖悬停才能阅读推荐原因。
- 屏幕阅读器应先读科技名、状态、理由、瓶颈，再读选择提示。

## 13. 降级与错误处理

- 元数据缺少 `effectTags`：保留 `general_growth`，降低推进优先级并在开发环境警告。
- 前置 ID 不存在或形成循环：距离设为 99，进入先处理的配置异常，不崩溃、不自动揭示。
- 价格资源状态不存在：库存和容量按零处理，标为不可达；不得当成免费。
- 流量分析不可用：只基于库存和容量判断，不显示等待时间。
- 解锁包存在未知类别：不计入解锁广度，但检查器仍按安全的已有格式展示。
- 无任何正式候选：显示三个有原因的空态；只有存在合法预览时才显示一个匿名远景。

## 14. 实现分层

- `game/data/definitions.js`：价格、依赖、路线、效果标签、解锁和可选受控提示。
- `game/systems/researchDecisions.js`：知识压力、档案、依赖距离、排序、多样性与队列快照；不得访问 DOM。
- `game/ui/render.js`：只消费 `ResearchQueueSnapshot`，复用研究签并显示理由/瓶颈。
- `game/ui/events.js`：维持选择、搜索、筛选和唯一研究操作，不含决策公式。
- `game/tests/researchDecisions.test.js`：固定夹具验证纯派生算法。
- `game/tests/research-list-runtime.html`：验证真实页面节点保持、选择和结构。

任何排序公式不得复制回 UI；任何价格、前置或状态不得从 DOM 文本反推。

## 15. 自动测试矩阵

| 类别 | 固定夹具 | 必须证明 |
| --- | --- | --- |
| 分段 | 可研究、缺资源、缺前置、已完成并存 | 三段正确、无重复、已完成不占槽 |
| 可达性 | 正流量、手动来源、无来源、容量不足 | 只有正流量可计算等待；硬阻断不进目标 |
| 多资源 | 两种货币不同缺口与流量 | 等待取最大值，主瓶颈取缺口比例 |
| 机会成本 | 当前目标与可研究支线争用同一货币 | 支线标记冲突、显示目标中文名与补回延迟，无冲突替代项优先 |
| 完成度 | 两个同档目标处于同一等待区间 | 完整价格已备齐比例更高者优先，避免低完成度目标抖动插队 |
| AND 前置 | 缺两个必需前置 | 状态受阻并指向稳定首个近端前置 |
| OR 前置 | 两项中完成一项 | 前置满足，不错误要求全部完成 |
| 依赖循环 | A→B→A | 不死循环，距离 99 并降级 |
| 推进价值 | 系统入口与普通同层强化 | 系统入口在无危机时优先 |
| 满仓建造阻断 | 菌菇 100/100，下一座菌菇床需约 121，采菌、采矿和大木棒均可研究 | 采菌排首位；理由具名连接储物坑、菌菇容量和菌菇床 |
| 多样性 | 同意图多项、其他意图与路线各一项 | 优先覆盖不同意图，再覆盖路线 |
| 保密 | unknown、preview 与正式项并存 | 未知不出现，预览不泄露正式价格/效果 |
| 不变量 | 对同一状态重复调用 | 签名与成员顺序完全一致且不修改状态 |

## 16. 交互验收矩阵

| 场景 | 预期 |
| --- | --- |
| 里程碑缺资源但持续可产 | 进入当前目标，显示真实主缺口和等待时间 |
| 高价值科技价格超过容量 | 不进入当前目标；进入先处理并显示需求与上限 |
| 账册或魔晶无取得路径 | 不显示伪等待；明确提示先开放来源 |
| 三项可研究属于同一增产意图 | 第二、三项优先覆盖其他意图和路线 |
| 可研究支线会花掉当前目标缺口资源 | 签面说明会占用哪个具名目标、多少资源及约延迟时间；无自动流量时明确不可自动补回 |
| 远期里程碑缺三层前置 | 推荐最近可行动铺路节点，不反复展示遥远目标 |
| 科技完成 | 立即退出队列，检查器仍保持该科技并显示已完成 |
| 输入中文搜索跨多个 tick | 原输入节点、焦点、选区和组合状态保持 |
| 中部光标编辑与删除 | 不跳到末尾，不丢字符，结果只在合并刷新后更新 |
| 打开浮窗或按住研究签 | 自动刷新不替换交互目标，松开/关闭后恢复正常刷新 |
| 键盘选择研究签 | 只更新选中态与检查器，队列 DOM 和滚动位置保持 |
| 暂停 | 队列成员保留、等待不推进、按钮显示“已暂停” |
| 预览进入空态远景 | 只显示匿名用途和具名前置，不显示价格、数值或后继 |

## 17. 当前实现同步基线

当前代码已经落实本文件的核心设计：

1. `game/systems/researchDecisions.js` 独立生成纯派生决策档案和三个队列区段，UI 不再保存或复制排序公式。
2. 可达性检查覆盖科技价格实际使用的全部资源；持续流量计算等待，手动来源说明主动取得，制作、贸易、掠夺和远征作为离散来源进入“先处理”，无来源与容量硬阻断不会伪装成等待目标。
3. AND/OR 前置、依赖距离、三层循环保护和最近可行动铺路均已接入；高价值远期目标会提高真实铺路节点的推进档，并显示具名目标。
4. 推进价值同时读取里程碑、解锁类别语义和直接解锁广度；主标签入口高于普通数组数量，经营意图覆盖优先于单纯路线去重。
5. 菌菇耗尽、住房触顶和生产劳力过载会产生生存或治理压力，使对应科技在危机时插队；军事、信仰、深渊和自动化继续由受控效果标签与解锁类别判断。
6. 普通成员变化采用三秒稳定窗口；研究完成、揭示、分段资格、容量和来源变化立即更新；保留成员仍刷新最新缺口与说明。
7. 队列签复用图鉴签的共享控件样式与选择行为，并在同一可聚焦按钮中追加推荐理由和唯一瓶颈；固定检查器仍拥有唯一研究按钮。
8. `game/tests/researchDecisions.test.js` 覆盖分段、去重、完成项排除、状态不变、持续/离散来源、容量、多资源等待、AND/OR、循环、铺路、压力、多样性、保密、确定性和防抖；真实 Chromium 用例覆盖输入节点、焦点、光标、目录节点保持和局部选择刷新。
9. 当前目标同时驱动资源保留诊断；可研究支线若挪用其缺口资源，会按冲突比例降序并显示可验证的补回延迟，严重经营危机可越过该保留规则。同档目标以完整价格备齐比例消除粗等待档内的低质量平局。
10. 已开放建筑若因满仓且下一座价格超过容量而不可达，系统会从结构化建筑容量效果反推直接解锁研究；该因果解法获得最高紧迫档，避免被无关科技的解锁数量压过。

后续增加新研究货币、取得系统、科技效果标签或解锁类别时，必须同步扩展本文件的封闭契约与自动夹具，不能退回解析中文文案或简单按价格、层级排序。
