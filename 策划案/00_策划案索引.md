# 哥布林帝国策划案索引

本文档集基于 `cat-zh` 中《猫国建设者》源码结构分析，重新设计为“哥布林帝国”题材的放置经营游戏。目标不是复刻文案与资产，而是吸收其多层资源、渐进解锁、职业分配、科技树、外交、宗教/威望和长线循环结构。

## 文档清单

1. `01_猫国建设者源码机制分析.md`
   - 说明源码中的核心系统、数据驱动方式、资源循环和长期成长结构。

2. `02_哥布林帝国核心策划案.md`
   - 定义游戏定位、世界观、目标体验、核心循环和阶段结构。

3. `03_资源建筑职业系统.md`
   - 给出资源表、加工链、建筑表、职业表及主要数值建议。

4. `04_科技外交信仰与重置.md`
   - 设计科技树、工坊升级、外交交易、祭祀信仰、威望重置和挑战模式。

5. `05_界面交互与开发落地规格.md`
   - 约定 UI 信息架构、暂停/继续、存档数据结构、解锁规则、数值公式和完整开发里程碑。

6. `06_玩法指南与新手教程.md`
   - 面向玩家说明界面、核心系统、阶段目标，并提供可直接照做的新手开局流程和常见问题排查。

7. `07_完整交互设计策划案.md`
   - 以当前运行时代码为准，完整规定全局框架、全部标签页、卡片、浮窗、点击响应、键盘响应、局部刷新、状态差分和逐页验收矩阵。

8. `08_建筑决策队列设计策划案.md`
   - 专项定义建筑决策队列的战略目标、瓶颈诊断、候选生成、确定性排序、防抖、解释文案、数据契约和验收案例。

9. `09_研究决策队列设计策划案.md`
   - 专项定义研究决策队列、完整研究图鉴、候选分段、路线多样性、刷新边界和验收案例。

## 设计基准

- 早期目标：从一小撮地穴流民发展为可稳定生产、挖矿、冶炼的氏族。
- 中期目标：建立地下城邦、军工体系、贸易网络和祭祀体系。
- 后期目标：扩张为地底帝国，研究符文机械、深渊远征和世界树根系之外的异界资源。
- 长线目标：通过帝国崩塌/迁徙重置获得永久威望，开启更深层科技和挑战玩法。

## 当前实现同步基准

截至当前 `game/` 静态应用，策划案和蓝图必须以这些运行时代码为真实设计来源：

- 静态 ID：`game/data/ids.js`。
- 静态定义、数值、解锁包：`game/data/definitions.js`。
- 中文显示文案：`game/data/text.js`。
- 存档和运行状态契约：`game/scripts/contracts.js`、`game/state/initialState.js`、`game/state/save.js`。
- 公式和主动系统：`game/systems/*.js`。
- 实际界面结构：`game/ui/render.js`、`game/ui/events.js`、`game/index.html`；建筑区与研究区均采用决策队列、完整图鉴与固定检查器组成的指挥板。
- 当前存档版本：`SAVE_VERSION = 32`。
- 当前模拟频率：`TICKS_PER_SECOND = 5`。
- 当前主标签页：地穴、氏族、研究、工坊、外交、祭祀、帝国、深渊；新手教程由顶部全局导航按钮打开。

本文档集中若旧段落使用“建议”“后续”“示例”并与上述实现基线冲突，以各文档新增的“当前实现同步基线”小节为准；旧段落只保留为设计演化背景，后续清理时应逐项压平。

## 命名原则

- 资源名称强调“粗粝、地下、掠夺、冶炼、祭祀”。
- 建筑名称强调“巢穴、矿坑、工坊、要塞、祭坛、深渊设施”。
- 科技名称从生存技巧逐步发展到军工、符文、深渊学和帝国制度。
- 不使用《猫国建设者》的猫、猫薄荷、独角兽等题材表达，仅保留系统骨架。
